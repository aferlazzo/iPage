About SpriteMiner

The spriteMiner web page was written out of frustration with the lack of graphic web development tools that provide a bulletproof way of obtaining the CSS code for an item within a sprite, particularly if you later want to make a change to a sprite.

In a perfect world one would come up with an idea for a web page, write the HTML code, the CSS, the javascript and server pieces, and then move on to the next assignment. But one then has to also assume that there are no specification mistakes, changes of direction, or anything else that could possibly go wrong. The real world of web development is not quite so tidy a place. For us, that means there are changes introduced by internal and external factors, if not during development then over time.

This web page helps in developing, maintaining, and improving HTML pages that use sprites. Notice the active-verb choices in the previous sentence. That's because there is rarely a 'development' or an 'improvement' event. Rather we are often developing and improving via a process called 'continuous improvement'. We make these changes because we begin with a vision of a HTML page, but one we achieve that vision we realize that an refinement to our original vision is necessary.

Along with the HTML code, the graphic objects and other formating decisions like paragraph placement and so forth are being fine-tuned. That's why the benefit of creating sprites during the initial web page development for page load speed and memory savings have the unwanted side-effect of making page 'tweaking' a more complex endeavor.

Creating a sprite at any point in the HTML page life-cycle results in efficiencies, but it also creates yet another thing that must be adjusted when a web page later changes. unfortunately there are few tools that help the person who is charged with making such a change to rebuild or adjust sprites, save for completely recreating the sprite from individual graphic files.

Help

Browse to the pathname of a sprite image file on the 'Sprite File' tab and click 'Upload'. It will appear within a grid. Next to it will appear a copy of the sprite without a grid. You can drag this copy around the page if needed for a better view of the selection.

To understand the sprite grid think of it as 2 pieces of transparent paper. The bottom piece, or layer, contains a copy of the sprite and the sprite background color. The top piece layer is the item you select and the selection tint.

Hover over a 'Selection Background' color to colorize the background the sprite is a light color in order to better see the sprite. Click on a color to click on it if it helps you see the sprite. Next, hover over a color on the 'Selection Background' tints the layer to make it easy to determine which part of the sprite is selected.

Initially the entire sprite will be selected and tinted but you can use the grid frame's coordinate numbers s as handles for adjusting the selection rectangle. Alternatively, you can directly adjust the selection by dragging the rectangle horizontally or vertically or by resizing it.

The 'Coordinates' tab is yet another place to adjust the sprite grid and the selected item on it.

Some sprites may be too large to view along with the selected item to the right. The sprite frame can be resized to make it easier to work on the portion of the sprite you are interested in. The top and left edges of the frame control the width and height of the visible sprite while the bottom and right frame edges allow the selected item to be adjusted.

After moving a handle click on one of the corners to reduce the sprite's size to the selected item's coordinates. A red corner means the height or width of the sprite is partially hidden. Clicking again will restore the sprite's height or width to full view.

The 'Item CSS' tab contains the css code to use once you have selected an item. This is an accurate representation of the item and is the result to use on your web pages.
